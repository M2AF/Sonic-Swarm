API ENDPOINTS (Quick Reference)
================================================================================

POST /api/resolve
  Input:  {artist, track, album}
  Output: {metadata, torrents[]}
  Purpose: Find torrents for a track

GET /api/search?q=query
  Input:  q (search query)
  Output: {results: albums[]}
  Purpose: Search for albums

POST /api/stream
  Input:  {magnet}
  Output: {streamId, status}
  Purpose: Start streaming a magnet

GET /api/swarm-stats
  Output: {activeTorrents, totalPeers, totalSeeds, ...}
  Purpose: Live P2P statistics

GET /api/health
  Output: {status, webtorrent, database, uptime}
  Purpose: Server health check

POST /api/prebuffer
  Input:  {magnets: []}
  Output: {prebuffered: []}
  Purpose: Pre-cache upcoming tracks

DEPLOYMENT OPTIONS
================================================================================

Option 1: Desktop (Single-Click)
  $ npm run build:installer
  Creates: SonicSwarm-Setup.exe
  Users download & install on their machine
  Perfect for personal use & privacy

Option 2: Self-Hosted Server (VPS)
  $ npm install --production
  $ npm start
  Access at: https://your-domain.com
  Perfect for community nodes

Option 3: Docker Container
  $ docker build -t sonicswarm .
  $ docker run -p 9191:9191 sonicswarm
  Perfect for orchestration & scaling

Option 4: Federated Network
  Multiple community nodes that gossip with each other
  Users connect to any node
  Metadata synced across network
  Perfect for large-scale decentralized system

See SETUP.md for detailed deployment guides.
